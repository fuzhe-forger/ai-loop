# 验收偏好

## 代码风格

- Bash 脚本必须有 `set -euo pipefail`
- Python 脚本必须有类型注解（后续推进）
- Markdown 文件必须通过 `--check` 检查

## 命名规范

- 脚本文件：`kebab-case.sh`
- 文档文件：`kebab-case.md`
- JSON 字段：`snake_case`

## 文档要求

- 每个脚本必须有 `--help`
- 每个阶段报告必须有"目标、已完成、验证结果、边界、结论"
- 每个设计文档必须有"定位、原则、MVP 范围"

## 测试要求

- 新增脚本必须有正向和负向测试
- 门禁脚本必须能正确失败
- 验证报告必须可重复生成

## 提交规范

- 单步提交：每个功能单独提交
- 提交信息：清晰说明做了什么
- 不混合：功能代码和文档分开提交
