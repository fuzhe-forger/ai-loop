# 架构约束

## C001: 本地优先

**约束**：AI 执行必须先在本地隔离环境运行，不能直接改生产。

**理由**：
- 降低风险
- 保留完整 evidence
- 人工复核后再回写

**适用范围**：所有 AI 执行任务

**检查点**：dry-run 必须通过才能进入真实执行

---

## C002: 证据优先

**约束**：每个 run 必须生成 core evidence，否则不能进入复核。

**理由**：
- AI 说完成不算，evidence 才算
- 失败必须可复盘
- 经验必须可沉淀

**适用范围**：所有 AI 执行任务

**检查点**：strict evidence gate 必须通过

---

## C003: 人控回写

**约束**：远端副作用（comment、status、metadata 写入）默认不自动执行，需要人工确认。

**理由**：
- 避免误操作
- 明确责任
- 可追溯

**适用范围**：所有涉及 Multica 回写的操作

**检查点**：writeback summary 必须明确记录是否执行和谁批准

---

## C004: 不存密钥

**约束**：不允许把 token、cookie、密钥写入 memory、runs 或 comment。

**理由**：
- 安全
- 避免泄露

**适用范围**：所有文档和 evidence

**检查点**：人工复核时检查敏感信息
