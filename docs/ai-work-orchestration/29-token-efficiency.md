# Token 使用率治理

## 1. 目的

把长任务的 token 使用从“靠习惯节省”升级为“可度量、可治理、可复盘”。目标不是越短越好，而是在事实准确、审批安全、evidence 完整的前提下，提高单位 token 产出的可验收结果比例。

## 2. 适用范围

适用于司南中的长 Loop、跨系统写回、方案设计、代码审查、运行验证和交接场景。短问答、一次性说明、必须完整披露的审批/安全/部署操作，不强制套用压缩策略。

## 3. 核心原则

- 先定位，后展开：先用 `rg`、标题、文件大小、摘要索引定位，再按需读取具体片段。
- 先引用，后复制：能用路径和行号引用 evidence 的，不复制全文。
- 先摘要，后原文：readback/fetch/json 默认先生成摘要，只在排错时展开字段。
- 先 handoff，后长聊：长任务阶段结束必须沉淀可复用 md，避免新会话吞完整历史。
- 审批不压缩关键边界：副作用、风险、回滚、部署、删除、权限变更必须完整清楚。
- 代码/命令/错误不改写：保留原始符号、路径、命令和错误文本。

## 4. 指标定义

| 指标 | 定义 | 采集方式 | 目标方向 |
|---|---|---|---|
| 输出 token 密度 | 输出中有效结论、操作、证据、下一步占比 | closeout 人工标注 + 后续脚本抽样 | 提高 |
| 上下文复用率 | 后续会话通过 md/evidence 继续，而非依赖完整聊天历史 | handoff 是否存在、是否被引用 | 提高 |
| 重复读取率 | 同一轮中重复整读同一大文件或同类快照次数 | audit 脚本统计 shell/history/artifacts | 降低 |
| Evidence 引用率 | 使用路径/行号/摘要引用 evidence，而非复制全文 | markdown 中路径引用和大段文本比例 | 提高 |
| 中间汇报压缩率 | 中间进展只保留状态、证据、下一步的比例 | 人工抽检 + 模板检查 | 提高 |
| 大文件展开率 | 超过阈值的 md/json/log 被整读的比例 | 文件大小 + 读取命令审计 | 降低 |
| Canonical snapshot 覆盖率 | 多版本 readback/fetch 中明确 canonical 文件的比例 | closeout 或 evidence index 检查 | 提高 |
| 估时-token 关系 | token 消耗、真实耗时、任务类型之间的关联 | 后续接入会话统计或人工样本 | 可解释 |

## 5. 阈值建议

| 对象 | 阈值 | 默认动作 |
|---|---:|---|
| markdown/json/log 文件 | > 12KB | 先 `rg`/headings/摘要，不整读 |
| readback/fetch/after JSON | 任意大小 | 先摘要字段和状态，不整读 |
| 技术方案/review pack | > 20KB | 先目录化，再按章节读取 |
| 进展汇报 | > 8 行 | 压缩成状态、证据、下一步 |
| handoff | > 120 行 | 改为索引式 handoff，引用原 evidence |

## 6. 输出策略矩阵

| 场景 | 输出策略 | 不可省略 |
|---|---|---|
| 低风险本地进展 | 强压缩 | 当前状态、证据路径、下一步 |
| 本地代码定位 | 压缩 | 文件、符号、行号、验证命令 |
| 文档/方案总结 | 中压缩 | 结论、边界、任务拆解、验收 |
| 飞书/Multica 写回草稿 | 轻压缩 | 写入目标、字段、内容、readback |
| 审批请求 | 不压缩关键边界 | 副作用、风险、回滚、是否部署 |
| 安全/删除/部署 | 不压缩 | 操作对象、不可逆性、确认语句 |
| Evidence | 摘要 + 路径 | 可追溯路径、生成时间、验证结果 |

## 7. Closeout Token 小节模板

每个长 Loop closeout 增加以下小节：

```markdown
## Token 使用复盘

- 大文件处理：是否整读 >12KB 文件；若有，原因是什么。
- Readback 处理：是否先摘要；canonical snapshot 是哪个。
- Evidence 引用：主要证据是否用路径/行号引用。
- Handoff：是否需要新会话；handoff 路径是什么。
- 浪费来源：重复读取/重复汇报/复制全文/无效等待。
- 下轮改进：下一轮减少 token 的具体动作。
```

## 8. Handoff 摘要模板

```markdown
# Handoff

## 当前目标

一句话说明目标和验收。

## 已完成

- 结果：
- 证据路径：

## 当前状态

- 分支/文件状态：
- 计时状态：
- 风险/审批状态：

## 下一步

1. 最小可验收动作。
2. 验证命令。
3. 停止条件。

## 不要重复读取

- 大文件：
- 已验证文件：
- canonical snapshot：
```
