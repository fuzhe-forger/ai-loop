# 司南 v1.1 Backlog

> 正式路线图见 `docs/ai-work-orchestration/product/sinan-v1-to-v2-roadmap.md`。本文件保留为 v1.1 近期执行候选池，不再作为完整路线图。

## 优先级说明

- P0：v1.0 使用中会直接影响稳定性或团队接入。
- P1：明显提升效率或可复制性，但不阻断 v1.0。
- P2：探索项，等治理数据更多后再做。

## Backlog

| 优先级 | 方向 | 目标 | 前置条件 | 风险 |
|---|---|---|---|---|
| P0 | 时间估计模型继续校准 | 提升 `<1 分钟` 命中率和同类任务估时准确性 | 更多 timestamp 样本 | 样本少时过拟合 |
| P0 | v1.0 E2E 案例补齐 | 补齐文档、代码、写回三类端到端案例 | 真实任务或沙箱任务 | 需要审批和脱敏 |
| P0 | Gate policy 体验优化 | 减少简单任务被重门禁拖慢 | v0.3 门禁使用数据 | 过度放松导致漏风险 |
| P1 | 服务化工作台 | 把文件协议逐步封装成稳定 CLI/API | 文件协议稳定 | 服务化过早增加维护成本 |
| P1 | 记忆索引增强 | 从 markdown 索引升级为可查询 KV/数据库 | 记忆分类稳定 | 记忆污染和召回误导 |
| P1 | 多仓库编排 | 支持任务跨多个 repo 执行和 evidence 聚合 | 单仓库闭环稳定 | 冲突和权限复杂度上升 |
| P1 | 飞书/Multica 字段适配器 | 对不同表结构做 schema mapping | v0.6 读写样例 | 字段变化导致写回失败 |
| P1 | 团队接入 doctor | 一键检查环境、权限、脚本、配置、skills | v0.9 手册稳定 | 环境差异多 |
| P2 | 策略自动化建议 | 根据历史 evidence 建议 gate 强度和审批范围 | 足够历史样本 | 不能自动替代人类审批 |
| P2 | 可视化看板 | 展示估时误差、返工率、gate 失败原因 | 数据标准稳定 | 看板变成额外维护负担 |

## 暂缓事项

- 自动 reviewer 最终裁决。
- 自动远端写回决策。
- 无审批部署。
- 把所有记忆直接写入长期库。
