# FUZ-554 Phase 1：首个案例复盘

## 原始需求

选择一个低风险试点，完整记录从 issue 到 evidence 的闭环案例。

## 目标

将该 Multica issue 转成本地 ai-loop 任务，先进行 dry-run 验证，再根据 summary 生成 comment 草稿。

## 验收

- 生成本地 task.md
- 成功运行 ai-loop dry-run
- 生成 summary.md
- 生成 multica-comment.md
- 默认不写回 Multica

## 安全边界

- 不自动改 issue 状态
- 不自动回写 comment
- 不自动 push / commit / MR
- 不访问生产系统
